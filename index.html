log-search
CLI · Node.js · 50GB in 0.3s
TypeScript Inverted Index CLI Watch Mode

A blazing-fast CLI search engine for massive log files. Builds an in-memory inverted index from your logs and queries 50 GB in under 0.3 seconds with AND/OR/NOT operators, regex, and fuzzy matching.

# Search 50GB logs in 0.3s
$ log-search "ERROR AND (timeout OR refused)"

# Regex pattern
$ log-search "/api\/v[0-9]+\/users.*500/"

# Fuzzy match + watch mode
$ log-search "~authentcation" --watch

# Exclude pattern
$ log-search "WARN NOT healthcheck" --tail 100
AND / OR / NOT operators

pdf-signature
Legal e-Signature SDK · Node.js
PKCS#7 eIDAS RFC 3161 X.509 KSA-ETL

A legally compliant PDF e-signature library. Produces PKCS#7 detached signatures with X.509 ephemeral certificates, RFC 3161 trusted timestamps (AdES-B-LT), and meets eIDAS, ESIGN, UETA, and KSA-ETL standards.

import { signPdf } from 'pdf-signature';

const signed = await signPdf({
  input: 'contract.pdf',
  output: 'contract.signed.pdf',
  reason: 'Approved by legal team',
  timestamp: true, // RFC 3161 TSA
  level: 'AdES-B-LT',
});

// ✅ PKCS#7 detached signature applied
// ✅ Timestamp token embedded (RFC 3161)
// ✅ eIDAS / KSA-ETL / ESIGN compliant

csv-super
Streaming CSV · Zero Memory · RFC 4180
Async Generator 0 deps RFC 4180 Backpressure TypeScript

Read a 10 GB CSV file without crashing RAM. Fixed ~50 MB memory footprint using native async generators and mechanical backpressure. Full RFC 4180 compliance, BOM detection, progress callbacks, and zero external dependencies.

import { csvSuper } from 'csv-super';

// RAM stays at ~50MB regardless of file size
for await (const { rows, totalSoFar } of
  csvSuper('data.csv', {
    batch: 1000,
    onProgress: ({ percentage, speedMBps }) =>
      console.log(`${percentage.toFixed(1)}% @ ${speedMBps} MB/s`),
  }))
{

Under the Hood

Architecture Principles

Every library follows the same engineering philosophy — predictable, composable, and production-safe.

⚡
Zero-Copy Streaming
csv-super uses async generators with native backpressure — no row accumulation, constant 50 MB heap regardless of file size.
🔒
Defense in Depth
ssrf-shield runs 5 sequential security layers: URL normalization → scheme → port → IP → DNS, each layer independently blocking a class of attacks.
📜
Legal Compliance First
pdf-signature is designed around eIDAS Article 26, RFC 3161 TSA, and PKCS#7 standards before any API decisions — compliance is not bolted on.
🔍
Inverted Index
log-search pre-builds a token → line-offset index at startup. 50 GB log queries resolve in 0.3s by jumping directly to matching byte offsets.
🧩
Clean Architecture
The Odoo View Builder follows strict data → providers → screens layering with Riverpod StateNotifiers — UI never touches persistence directly.
🎯
100% TypeScript
All 4 Node.js packages ship with strict TypeScript, zero any in public APIs, and inferred generics — no @types package needed.
